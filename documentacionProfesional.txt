1. Resumen ejecutivo
Aplicación web para la gestión de carta, mesas y reservas de Tsinghe Fusion. Soporta dos roles principales: `admin` y `comensal`. Provee panel administrativo y vistas públicas (menu, reserva).

2. Objetivos del proyecto
- Digitalizar reservas y gestión de mesas.
- Centralizar la gestión del menú (platos, categorías, alérgenos).
- Separar permisos por rol y asegurar flujos críticos (reserva, asignación de mesas).

3. Stack tecnológico
- Frontend: React (JSX), hooks, `react-router-dom`.
- Estilos: Tailwind + CSS en `src/styles`.
- Backend: Firebase (Auth, Firestore, Storage, Hosting, Functions).
- Build: Vite. Node.js (16+ recomendado).

5. Componentes y responsabilidades (clave)
- `src/pages/Menu.js`: vista pública del menú, carga categorías, platos y alérgenos; controla CTA según rol.
- `src/pages/Dashboard.js`: enrutamiento interno del panel; interpreta `?section=` para renderizar secciones.
- `src/components/ReservationForm.js`: validaciones y envío de reservas desde comensales.
- `src/components/AdminReservationForm.js`: creación/edición de reservas por admin.
- `src/components/Sidebar.js`: lista las secciones disponibles según rol.

6. Servicios y APIs internas
- `AuthService.js`: login, registro, manejo de token/roles.
- `ReservationService.js`: CRUD de reservas, validación de disponibilidad.
- `TableService.js`: CRUD de mesas y helpers de capacidad.
- `MenuService.js`: operaciones sobre platos, categorías y alérgenos.
- Todos los servicios devuelven respuestas uniformes `{ success, data, error }`.

8. Flujo de autenticación y autorizaciones
- `AuthContext` inicializa sesión y mantiene `user` y `role` en contexto.
- Protecciones de ruta: `Dashboard` y hooks como `useDashboard` filtran las vistas disponibles según `role`.
- Redirecciones: intentos de reserva por usuarios no autenticados van a `/login?next=/dashboard?section=reservas`.

9. Navegación y rutas importantes
- `/` — Inicio.
- `/menu` — Carta pública.
- `/login` — Login (soporta `next` para redirección post-login).
- `/dashboard?section=inicio` — Panel admin.
- `/dashboard?section=reservas` — Reservas comensal.
- `/dashboard?section=admin-reservas` — Todas las reservas (admin).

10. Integración con Firebase
- Archivo clave de inicialización: `src/firebase/index.js`.
- Reglas: `firestore.rules` debe limitar escrituras a colecciones sensibles (p.ej. `plates`, `tables`, `reservations`) a usuarios con rol `admin` cuando corresponda.
- Cloud Functions: usar `functions/` para tareas asíncronas: notificaciones, limpieza de `no-show` y sincronización de datos.

11. Configuración del entorno local (detallada)
- Requisitos mínimos: Node.js 16+, npm 8+.
- Variables de entorno (ejemplo `.env.local`):
  VITE_FIREBASE_API_KEY=...
  VITE_FIREBASE_AUTH_DOMAIN=...
  VITE_FIREBASE_PROJECT_ID=...
  VITE_FIREBASE_STORAGE_BUCKET=...

Pasos para desarrollo:
1. Clonar repo.
2. Copiar `.env.example` a `.env.local` y completar variables.
3. `npm install`.
4. `npm run dev` — inicia Vite y sirve la app en `localhost`.

Usando emuladores Firebase (opcional):
1. `firebase login`
2. `firebase use --add` (seleccionar proyecto)
3. `firebase emulators:start` (emula Auth, Firestore, Functions)

12. Construcción y despliegue (comandos)
- Desarrollo:
```bash
npm install
npm run dev
```
- Build producción:
```bash
npm run build
```
- Despliegue a Firebase Hosting y Functions:
```bash
firebase deploy --only hosting,functions
```

13. Buenas prácticas y recomendaciones
- Normalizar fechas y horas en UTC en DB y formatear en cliente según zona local.
- Usar transacciones Firestore para asignación de mesas y evitar race conditions.
- Validar inputs tanto en frontend como en reglas/Functions.
- Añadir logging y métricas en Functions para auditoría.

14. Extensión y mantenimiento
- Añadir tests unitarios con Jest y e2e con Playwright/Cypress para flujos críticos.
- Scripts de migración en `scripts/` para cambios en esquemas.

15. Problemas conocidos y soluciones
- Doble asignación de mesa: usar transacción + marca temporal.
- Zonas horarias: almacenar en ISO 8601 y convertir en frontend.
